Products on Crash Stats

The Socorro crash ingestion pipeline handles incoming crash reports for Mozilla products. Crash reports are collected, processed, and available for search and analysis using the Crash Stats web site.

Incoming crash reports have a ProductName key which specifies which product the crash report is for. For example, incoming crash reports for Firefox desktop have Firefox as the key. This corresponds to the MOZ_APP_NAME value which is set when building the product.

Crash reports for products that Socorro doesn't recognize are discarded.

Socorro collects build information for products. Currently, build information is scraped from archive.mozilla.org. In the future, we plan to switch to using BuildHub.

Build information is required to:

  • fix version numbers in processing
  • drive the product home page in the Crash Stats site
  • support forms in the Top Crashers page in the Crash Stats site
  • support forms in the Exploitability page in the Crash Stats site
  • suggest products in the "Product" field in the Super Search page in the Crash Stats site

If your product doesn't have build information on archive.mozilla.org, then parts of the site won't work for your product.

Socorro collects active version information from product-details.mozilla.org.

Current active version information is required to:

  • drive the product home page in the Crash Stats site
  • support forms in the Top Crashers page in the Crash Stats site
  • support forms in the Exploitability page in the Crash Stats site

If your product doesn't have active version information on product-details.mozilla.org, then parts of the site won't work for your product.

How to request a new product on Socorro

To request a new product be added, please file a bug report.

In the bug report, please specify the following things:

  1. The name of the product as you want it to show up in the Crash Stats web interface.
  2. The MOZ_APP_NAME used when building the binaries for the product. This is the value of the ProductName field in incoming crash reports.
  3. (Optional) The URL for build information for your product--parts of the site won't work without it.
  4. (Optional) The URL for active version information--parts of the site won't work without it.

If you have any questions or additional needs, please email us.
